What is the complexity of the algorithm ?

  • Time Complexity

    • Radix Sort sorts numbers one digit at a time. The time it takes depends on:

    • n: The number of elements to sort.
    • d: The number of digits in the largest number.
    • k: The range of values each digit can take (like 0-9 for decimal numbers).

    The total time to sort is O(d * (n + k)). This means we do d rounds of sorting, and each round takes O(n + k) time.

    For example, if you are sorting 100 numbers, each with at most 3 digits (like 457, 234, etc.), and you're using decimal numbers (0-9), it will take 3 rounds of sorting.

  • Space Complexity

    • Radix Sort uses extra space to hold the sorted output at each step and to count how many times each digit appears. This extra space is proportional to:

    • n: The number of elements.
    • k: The range of possible digit values (0-9 for decimal numbers).

    So the space complexity is O(n + k).
